# E-Commerce Pricing Benchmarking: Quick Commerce vs. Standard Delivery

## Project Overview
This project analyzes and compares the pricing models across 50+ e-commerce listings to identify profitability gaps and consumer cost trends. Specifically, it benchmarks rapid-delivery (10-minute quick commerce) margins against standard delivery (45-minute food delivery) models. 

The goal of this analysis is to evaluate pricing capabilities across distinct delivery business lines, visualize data trends, and propose strategic product enhancements based on fee-to-item-price ratios.

## Tech Stack & Tools
* **Language:** Python
* **Libraries:** Pandas, NumPy
* **Concepts:** Data Requirement Analysis, Competitive Benchmarking, Margin Optimization, GTM Strategy Basics

## Key Objectives & Analysis
1. **Total Cost Calculation:** Aggregated base prices, delivery fees, platform fees, and surge multipliers to find the true cost to the consumer.
2. **Delivery Premium Benchmarking:** Calculated the "Fee Percentage" (what portion of the user's total bill goes strictly to delivery and platform fees).
3. **Profitability Gap Identification:** Grouped data by delivery model to highlight items where markup and delivery costs outpace standard profitability curves.

## Key Insights
* **High-Markup on Low-Value Carts:** Quick commerce models show a significantly higher fee-to-item-price ratio on basic staples compared to standard restaurant delivery.
* **Margin Opportunities:** Identified specific product categories where dynamic pricing and bundled delivery fees could optimize platform margins without deterring user conversion.

## How to Run
1. Clone this repository.
2. Ensure you have Python installed along with the `pandas` and `numpy` libraries.
3. Run `pricing_analysis.py` to generate the mock dataset and view the benchmarking results in your console.
